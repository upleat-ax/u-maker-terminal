U-MAKER Plugin

A PDCA-based SSoT (Single Source of Truth) collaboration orchestrator.

A Claude Code plugin that automates the entire software development lifecycle with 4 specialized agents and a 3-layer pipeline.

Plugin v3.0.10 SSoT Config v3.0.0 35 Skills 4 Agents

TL;DR

Forward Engineering (New Project)

# 1. Install (macOS / Linux)
curl -fsSL https://raw.githubusercontent.com/thinoo-v2/u-maker-plugin/main/install.sh | bash

# 2. Restart Claude Code, then run in your project
/u-init my-app                # Create .u-maker/ structure + register app

# 3. Collect data → Analyze → Plan
# → Drop RFP, meeting notes, AS-IS materials into _input/raw/ (auto-classified)
/u-ingest [app]              # raw → classified analysis
/u-plan [app]                # Auto-generate SRS + IA + Roadmap

# 4. Design → Build → Verify
/u-design [app]              # ERD + API + Screen + Flow + RTM + wireframes
/u-dev [app]                 # Generate FE + BE + DB code
/u-qa [app]                  # TC design + test execution + report

# 5. Ship + Retrospective
/u-ship [app]                # Final verification + iteration log + retrospective

Reverse Engineering (Existing Project)

/u-init my-app                # Initialize project
/u-reverse [app]             # Source code → SSoT reverse engineering (ERD, API, Screen, SRS, etc.)
/u-sync [app]                # Cross-document consistency verification

Natural Language Also Works

"Create an SRS for the [app]"      # The u-maker router auto-dispatches

1. What This Plugin Solves

U-MAKER implements intent-driven, document-centric development (SSoT) as a collaboration orchestrator.

Traditional Approach vs U-MAKER v3

Traditional ApproachU-MAKER v3 Approach
Code first, documentation laterDocs first, code generated from docs (Docs-First)
Existing projects have no documentation/u-reverse auto reverse-engineers source code into SSoT
Analyzing a 200-page RFP at once → early content lost3-Layer pipeline with chunk analysis → zero information loss
"Which documents should I create?"One /u-plan [app] auto-generates SRS + IA + Roadmap
Requirements are untraceable4-Tier IDs (USR→FR→US→FT) enable end-to-end traceability
Editing SRS requires manual ERD/Screen/TC updates_links.json dependency graph enables Auto-Cascade
Skipping phases degrades qualityPhase Gate auto-verification before proceeding to next phase
File classification is tediousDrop into _input/raw/ for auto-classification (rfp/, as-is/, etc.)
Document and code versions diverge.md + .json generated simultaneously
Different rules per appcommon/ policy inheritance + per-app overrides
AI decisions are a black boxAssumptions Log records every decision + post-review
Document language is fixedlanguage.documents setting supports ko/en/ja/zh
Design artifacts hard to visualizewireframes/index.html SPA viewer auto-generated
Writing PR/MR is repetitive work/u-git-pr auto-generates structured PR/MR
Don't know where to ask questions/u-ask for lightweight Q&A

4. 3-Layer Data Pipeline

Processing raw data all at once causes early content to be lost due to context window limits. An intermediate refinement layer solves this:

_input/ (Raw) → _classified/ (Structured JSON) → docs/ (Deliverables) raw/ requirements/, workflows/ SRS, ERD, Screen rfp/ pain-points/, screens/ API, TestCase as-is/ decisions/, constraints/ Roadmap, RTM interviews/ domain-terms/, stakeholders/ wireframes/index.html

Auto-Classification in _input/raw/

When files are dropped into _input/raw/, running /u-ingest analyzes filenames and content, automatically classifying them into appropriate subfolders such as rfp/, as-is/, meeting-notes/, etc.

6. Commands

Grammar

/u-{command} [scope] [target] [flags]
  • scope: app name | common | all | omitted (auto-select)
  • target: document/item name
  • flags: -i (interactive) | --step | --only X | --cascade | --dry-run | --json

8. Interaction Modes

ModeFlagBehavior
auto(default)Runs without questions. Decisions recorded in Assumptions Log
interactive-iPauses only at decision points. Everything else runs at auto speed
step--stepPauses + requires approval at every step

Assumptions Log

A safety mechanism that records every time an agent "decides instead of asking" in auto mode.

/u-status [app] --assumptions    # Review pending assumptions
/u-assume approve A-001           # Approve
/u-assume reject A-001 "full cancel only"  # Reject → cascade modification

When maxAssumptions (default: 20) is exceeded, the system automatically switches to interactive mode.

9. 4-Tier ID Hierarchy

USR-0001 (User Type: Admin, General User) +-- FR-0001 (Functional Req: User Registration Feature) +-- US-0001 (User Story: I want to register with email) +-- FT-0001 (Feature = Implementation Unit: Email Verification)
FT = Feature (implementation unit). Not Functional Test.

Config Settings

{
  "language": {
    "documents": "ko",
    "supported": ["ko", "en", "ja", "zh"]
  }
}

Set the deliverable document language with language.documents. Supports ko (Korean), en (English), ja (Japanese), zh (Chinese). IDs and code are always in English.

wireframes/index.html

An SPA viewer auto-generated when /u-design runs. Navigate Screen documents via sidebar navigation and visualize wireframes with MD rendering.

Scope-First Navigation

Claude reads only _index.json first (instead of opening files directly), then selectively loads only the needed files -- saving context window.

Auto-Cascade

When SRS changes, impact flags automatically propagate to ERD, Screen, and TestCase via _links.json. Use /u-update --cascade to auto-update dependent documents.

13. Hooks & Guardrails

All hooks are bound to the PostToolUse(Write) event and execute automatically after file writes.

ScriptBehavior
on-input-added.jsDetects _input/ file → suggests /u-ingest
on-doc-change.jsCascade impact notification via _links.json
on-classified-validated.js_classified/ item validated → suggests next step
on-session-wrap.jsSession complete → notifies _classified loading
on-build-complete.jsBuild completion detected → suggests /u-qa
on-check-fail.jsTest failure detected → suggests backlog registration
on-gate-pass.jsGate condition met → suggests phase transition
on-ship-incomplete.jsIncomplete ship items → suggests carry-over to next iteration

SymptomSolution
